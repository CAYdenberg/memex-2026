Security is going to be something we'll have to clean up.

Since the client generally talks directly to firebase, as it stands right now, users generally have access to each other's stuff, as long as they have the other user's ID. Those aren't that hard to get, and since they don't change, any stolen access will remain stolen.

What we should do is move to a system where data is gated by the API instead of by Firebase. That way we can account for subscriptions, multi-tenancy, and admin-level access in a place that is fully controlled by us.

The thing that's worth thinking about is how you want access gated. Questions like:
- In a group subscription, can all the users in the group see each other's files?
- If a subscription lapses, can the users still get to their old files?
- If a user is removed from their subscription, can they get to their old files?
